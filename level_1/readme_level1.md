# Level 1 — SPI Loopback

**Status:** Working end-to-end with own bitstream (July 18, 2026)

Proves two-way SPI communication between the RP2040 and the SLG47910V FPGA on Shrike Lite: own `main.v` + Vicharak's `spi_target.v` → GCSH synthesis → own `spi_loopback_led.bin` → PIO firmware → SPI0 on RP2040 ↔ FPGA GPIO3–6 → byte echo + LED control from received data.

## Pin assignments (GCSH I/O Planner)

| Port | FPGA pin | Connected to RP2040 pin |
|---|---|---|
| clk | OSC_CLK (internal 50 MHz) | — |
| clk_en | OSC_EN | — |
| rst_n | GPIO18 (Pin 9) | GPIO14 |
| spi_sck | GPIO3 (Pin 16) | GPIO2 |
| spi_ss_n | GPIO4 (Pin 17) | GPIO1 |
| spi_mosi | GPIO5 (Pin 18) | GPIO3 |
| spi_miso | GPIO6 (Pin 19) OUT | GPIO0 |
| spi_miso_en | GPIO6 (Pin 19) OE | — (tri-state control) |
| led | GPIO16 (Pin 7) OUT | — |
| led_en | GPIO16 (Pin 7) OE | — |

## Flash procedure

1. Open the PIO project in VS Code with PlatformIO.
2. Place `spi_loopback_led.bin` (renamed from `FPGA_bitstream_MCU.bin` generated by GCSH) in `data/`.
3. Put board in BOOTSEL mode (hold BOOT, plug USB, release BOOT). Board mounts as `E:` drive.
4. PlatformIO → Project Tasks → pico → General → **Full Clean**. Required to avoid stale LittleFS baked into the previous UF2.
5. PlatformIO → Project Tasks → pico → Platform → **Build unified FW+FS UF2 image**.
6. Copy the resulting `firmware_with_fs.uf2` from `.pio/build/pico/` into the `E:` drive.
7. Board reboots. Open Monitor on the new COM port.

## Expected serial output

```
SPI Loopback Example
[ShrikeFlash] Flashing: /spi_loopback_led.bin
[ShrikeFlash] File size: 46408 bytes
[ShrikeFlash] FPGA programming done.
SPI initialized. Starting loop...
Sent 0xAB, Received 0x00
Sent 0xFF, Received 0xAB
Sent 0xAB, Received 0xFF
Sent 0xFF, Received 0xAB
```

Off-by-one echo is by design (see below). LED goes ON on 0xAB, OFF on 0xFF.

## Lessons learned this session

1. PlatformIO caches the LittleFS partition inside the compiled UF2. After renaming or replacing files in `data/`, always run **Full Clean** before **Build unified FW+FS UF2 image**. Otherwise the flashed UF2 still contains the old filesystem layout and `shrike.flash()` returns "File not found".
2. SPI transactions must be wrapped individually in earlephilhower's arduino-pico core: `SPI.beginTransaction() … SPI.transfer() … SPI.endTransaction()` around EACH transfer, not left open across the whole program.
3. Windows Explorer rounds file sizes to KB. 46408 bytes shows as "45.3 KB". Can't compare bitstreams via Explorer's display — use `dir` in PowerShell or file Properties for actual byte counts.
4. Off-by-one echo in `spi_target.v` is expected, not a bug. `tx_data_reg` updates at end of transaction N and is transmitted during transaction N+1. The first-ever transaction returns 0x00 (initial value of `tx_data_reg`).

## What is NOT proven yet

- Multi-byte transactions
- A higher-level command protocol on top of SPI (e.g. `[cmd][target_rate_hi][target_rate_lo]` for the motion coprocessor)
- Motor control RTL running on the FPGA
